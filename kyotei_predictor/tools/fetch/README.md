## 運用ルール
- データ取得は必ずraw/に保存し、命名規則を統一
- エラー処理・レート制限・API仕様変更に注意
- 不要な一時ファイルは随時削除

### CLI引数・出力・エラー処理の統一方針
- コマンドライン引数（例: 日付・会場・レース番号）はargparse等で指定可能に統一
- 出力ファイルはdata/raw/やoutputs/に保存し、命名規則を統一
- 成功・失敗時の標準出力メッセージも統一
- 例外発生時は明確なエラーメッセージ＋exit codeで終了
- ログ出力やリトライ処理はtools/common/の共通機能を活用

#### 代表的なCLI例
```bash
python race_data_fetcher.py --date 2024-07-01 --stadium KIRYU --race 1
```

#### 出力例
- data/raw/race_data_20240701_KIRYU_R1.json
- 成功: "✅ データ取得成功: ..."
- 失敗: "❌ エラー: ..."

# Fetch Tools

競艇データ取得ツール群です。公式サイトからのデータスクレイピングを担当します。

## 📁 ファイル構成

- `race_data_fetcher.py` - レースデータ取得（出走表・結果・条件等）
- `odds_fetcher.py` - オッズデータ取得（3連単のみ。2連単・2連複・3連複は取得できないため使用しない）
- `fetch_new_race.py` - 新規レース取得（統合ツール）

## 🚀 使用方法

### レースデータ取得
```bash
python race_data_fetcher.py
```

### オッズデータ取得
```bash
python odds_fetcher.py
```

### 新規レース取得（統合）
```bash
python fetch_new_race.py
```

## 📊 取得データ

### レースデータ
- 出走選手情報（名前・級別・成績等）
- 機材情報（ボート・モーター番号・成績等）
- レース条件（天候・風速・水温等）
- レース結果（着順・タイム・スタート等）

### オッズデータ
- 3連単オッズ（120通り）
- 3連複オッズ（20通り）
- 各種組み合わせの払戻金

## 🔧 技術仕様

- **データソース**: 公式サイト（metaboatraceライブラリ使用）
- **出力形式**: JSON
- **エラーハンドリング**: 例外処理・リトライ機能
- **レート制限**: 適切な間隔でのリクエスト制御
