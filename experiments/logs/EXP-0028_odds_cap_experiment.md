# Experiment: EXP-0028 odds cap experiment

## experiment id

EXP-0028

## purpose

高 EV 帯の損失原因を特定する一環として、オッズが cap を超える組み合わせをベット対象外にする odds_cap を none / 30 / 50 / 80 で比較する。高オッズ（人気薄）が損失の主因かどうかを検証する。

## background

- EXP-0027 で高 EV 帯（top 1〜20%）が損失源と判明。高 EV レースでは高オッズの組み合わせが多い可能性がある。
- オッズキャップにより高オッズベットを除外した場合に ROI が改善するか検証する。

## command

```bash
PYTHONPATH=. python3 -m kyotei_predictor.tools.run_odds_cap_experiment \
  --db-path kyotei_predictor/data/kyotei_races.sqlite \
  --n-windows 12 \
  --output-dir outputs/confidence_weighted_sizing_experiments
```

## interpretation

- **ROI**: 全戦略で **none（キャップなし）が最良**。odds_cap=30/50/80 はいずれも none より ROI が悪化（-14%〜-18%）。
- **bet_count**: キャップをかけるとベット数が減少（30 で約 6,300〜6,600、50 で約 8,200〜8,700、80 で約 10,000〜10,600）。
- **結論**: 高オッズを除外しても ROI は改善せずむしろ悪化。高 EV 帯の損失原因は「高オッズの組み合わせ」だけでは説明しきれない。キャップ後も残るベットの成績が相対的に悪い、または高オッズ以外の要因（キャリブレーション・選別ロジック）が主因の可能性。

## best cutoff candidate

- 本実験では **odds_cap による ROI 改善は得られず**。best candidate は **none（キャップなし）** のまま。adopt するキャップはなし。

## conclusion

- オッズキャップ（30/50/80）は ROI を悪化させた。高 EV 帯の損失原因として「高オッズ」を単純に除外する方針は有効でなかった。
- 採用判断: **reject**（odds_cap の採用は見送り）。高 EV 帯の損失要因の切り分けには、オッズ以外の軸（キャリブレーション・EV 閾値・レース属性など）の検証が有効と考えられる。

## next action

- 現行ベストは odds_cap なしのまま。高 EV 帯の損失原因のさらなる切り分けは、EV percentile 除外（EXP-0027 の次実験候補）やキャリブレーション改善で検証する。
