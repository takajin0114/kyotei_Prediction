# EXP-0058: CASE6 + CASE2 の stake 最適化

## 目的

CASE6 : CASE2 の stake 比率を 100:0 〜 100:100 で変化させ、**最大 profit / 最大 ROI / 最良 profit_per_drawdown** を評価し、CASE6 単体を標準維持するか、CASE6 + CASE2 の stake 最適を新標準にするかを判断する。

## 条件

- **比較**: C6:C2 = 100:0, 100:10, 100:20, 100:30, 100:40, 100:50, 100:75, 100:100（8 条件）
- **固定**: 同一予測・窓・switch_dd4000。重複ベット時は stake 加算（(odds, hit) キーでマージ）
- **n_windows_list**: 24, 30, 36

## 実行コマンド

```bash
python3 -m kyotei_predictor.tools.run_exp0058_stake_optimization --n-windows-list 24,30,36
```

## 解釈

- **利益・ROI・効率**: CASE2 比率を上げるほど profit・ROI・profit/dd は単調に改善し、100:100 がすべての指標で最良。
- **リスク**: C6_C2_100_75 と C6_C2_100_100 は **longest_losing_streak = 14** となり、100:50 以下の 5〜6 と比べて連敗が長い。max_drawdown も 100:0 の 12,618 から 100:100 で 18,882 まで増加。
- **トレードオフ**: 100:50 は profit 17,560・longest_lose 6・max_dd 15,750 で、EXP-0057 の「CASE6 + CASE2 50%」と整合。100:100 は利益最大化だが連敗・DD を許容する必要がある。

## 採用判断

- **結論**: **実運用標準は当面 CASE6 単体（C6_C2_100_0）のまま維持する。** CASE6 + CASE2 の stake 最適（100:100）は **攻め用の上限オプション** として扱い、資金・リスク許容度に応じて 100:50 や 100:100 を選択する。
- **新標準に昇格するか**: 数値上は **C6_C2_100_100 を新標準にすると profit・ROI・profit/dd は最大** となるが、longest_losing_streak 14 と max_drawdown 約 1.9 万は実運用で負荷が大きい。**標準＝安定軸** を維持するなら CASE6 単体のままが無難。**利益を最優先** し連敗・DD を許容する場合は、CASE6 + CASE2 100:100 を新標準として採用することは可能。
- **推奨**: 標準＝CASE6 単体、攻め用＝C6_C2_100_50（バランス）または C6_C2_100_100（最大利益・高リスク）の 2 本立てを推奨。
