# Experiment Status Rules

実験の **status** と **decision** の意味と運用ルールを明文化する。

## status

| status | 意味 |
|--------|------|
| planned | 実験が計画されている。まだ実行していない。 |
| running | 実装または実行を開始した。結果はまだ未確定。 |
| completed | 実行が終わり、結果が確定した。 |
| rejected | 実験を却下した（実行しなかった、または実行したが採用しない）。 |

## decision

結果が出た実験についての採用判断。

| decision | 意味 |
|----------|------|
| adopt | 採用する。基準や本番候補にする。 |
| hold | 保留。いったん採用しないが、再評価の可能性あり。 |
| reject | 却下。採用しない。理由は本文に書く。 |
| pending | まだ判断していない。 |

## 運用ルール

1. 実験開始前に、実験ログを **planned** で作成する。
2. 実装または実行を開始したら **running** に更新する。
3. 結果が確定したら **completed** に更新し、必要に応じて **decision** を記入する。
4. 採用しない実験も **rejected** または **decision=reject** で残す。再実行を防ぐため。
5. AI が rejected 実験に類似した提案をする場合は、**差分を明示**する。
