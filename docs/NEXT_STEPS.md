## Phase 4-1: 期待値閾値最適化 要件・設計方針

### 目的
- 投資判断の基準となる「期待値」の最適な閾値を探索し、リスクとリターンのバランスを最適化する
- 複数の投資戦略（保守的・バランス・積極的）に応じた最適閾値を決定する

### ゴール
- 過去データを用いた閾値ごとのパフォーマンスシミュレーション
- 最適閾値の自動探索アルゴリズムの実装
- 各戦略ごとのパフォーマンス比較レポート作成

### 入力データ
- 予測確率付きの3連単組み合わせデータ（例: outputs/trifecta_dependent_model_*.json）
- 実際のオッズデータ（例: outputs/real_odds_investment_*.json）

### 出力仕様
- 閾値ごとの投資件数・的中率・回収率・リスク指標（最大ドローダウン等）
- 最適閾値とその時のパフォーマンス指標
- 戦略ごとの比較表・グラフ

### 評価指標
- 的中率
- 回収率（総払戻/総投資）
- 最大ドローダウン
- シャープレシオ等のリスク指標

### 実装方針
- 閾値を0.8～2.0程度でグリッドサーチし、各閾値でのパフォーマンスを集計
- 保守的（期待値>1.5）、バランス（>1.2）、積極的（>1.0）など複数戦略で比較
- 結果をCSV/グラフで出力
- コードはtools/analysis/配下に新規スクリプトとして実装予定

## モデル精度向上タスク: 失敗パターンの深掘り（A）

### 目的
- 予測的中しなかったレース（失敗レース）の特徴量分布を分析し、外れやすい条件や傾向を可視化・定量化する。
- レース前に取得できるデータ（天候、モーター性能、選手級別、オッズ等）ごとに、的中/外れで分布比較し、改善ポイントを抽出する。

### 具体的なタスク
1. **失敗サンプル（miss_samples_*.json）から失敗レースのファイル名リストを抽出**
    - 失敗レースの特徴量（天候、モーター性能、選手級別、オッズ等）をpandas等で集計。
2. **特徴量ごとに的中率・外れ率を分布比較**
    - 的中/外れで各特徴量の分布を比較（棒グラフ・箱ひげ図など）
    - 例: 天候別、モーター性能別、人気別で的中率を可視化
3. **外れやすい条件や傾向をレポート化**
    - 分析結果をまとめ、外れやすいパターンや改善のヒントを抽出
4. **分析結果・気づきを本ドキュメントやREADMEに追記**
    - 再現性のある改善サイクルとして記録

### 【実施手順】失敗レースの特徴量分布分析・可視化

1. 失敗サンプル（miss_samples_*.json）から失敗レースのファイル名リストを抽出
2. race_data_*.json（kyotei_predictor/data/raw/）から該当レースの特徴量データを取得
3. レース前に取得できる特徴量（天候、モーター性能、選手級別、オッズ等）ごとに、
   - 的中/外れで分布比較（棒グラフ・箱ひげ図など）
   - 外れやすい条件や傾向を可視化
4. 分析結果・気づきを本ドキュメントやREADMEに追記
