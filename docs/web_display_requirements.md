# Web表示機能 要件定義書

## 1. 基本要件

### 1.1 システム概要
- **システム名**: 競艇予測結果表示システム
- **目的**: 機械学習による競艇予測結果を、直感的で使いやすいWebインターフェースで表示
- **対象**: 競艇予測システムの利用者（現状は自分のみ。今後拡張の可能性あり）

### 1.2 ターゲットユーザー
- **プライマリユーザー**: 競艇予測システムの利用者（現状は自分）
- **セカンダリユーザー**: システム管理者・開発者（現状は自分）

### 1.3 主要ユースケース

#### UC-001: 予測結果の確認
- **アクター**: 競艇ファン・投資家（現状は自分）
- **目的**: 最新の予測結果を確認し、3連単の予想確率とオッズを比較して購入判断を行う
- **前提条件**: 予測システムが実行済み、予測データが生成されている
- **基本フロー**:
  1. Webページにアクセス
  2. 集計サマリーを確認（対象会場数、予測レース数など）
  3. 興味のある会場・レースを選択
     - **トップページ**: 本日分の予測を確認するページ（タブ）
     - **過去ページ**: 過去の予測結果を確認するページ（タブ）
  4. 詳細な予測結果（3連単上位20組、購入提案）を確認
  5. 購入判断の参考として活用

#### UC-002: フィルターによる絞り込み
- **アクター**: 競艇ファン・投資家（現状は自分）
- **目的**: 特定の条件に合う予測結果のみを表示
- **基本フロー**:
  1. 会場フィルターで特定会場を選択
  2. リスクレベルフィルターでリスクを絞り込み
  3. 条件に合うレースのみを表示
  4. 詳細結果を確認

#### UC-003: システム状況の確認
- **アクター**: システム管理者・開発者（現状は自分）
- **目的**: 予測システムの実行状況・性能を確認
  - **専用ページ**も用意する
- **基本フロー**:
  1. 集計サマリーで実行状況を確認
  2. モデル情報・学習データ期間を確認
  3. 予測精度・成功率を評価
  4. システム改善の参考として活用

### 1.4 非機能要件

#### 1.4.1 パフォーマンス要件
- **レスポンス時間**: ページ読み込み3秒以内
- **同時接続数**: 100ユーザーまで対応
- **データ更新頻度**: 予測実行後リアルタイム反映
  - **オッズデータ**: レースこれから分のオッズデータは画面からボタン押したタイミングで取得する

#### 1.4.2 可用性要件
- **稼働率**: 99%以上
- **保守時間**: 予測実行時間以外は24時間稼働
- **障害復旧**: 30分以内

#### 1.4.3 セキュリティ要件
- **認証**: 現時点では不要（公開情報として扱う）
- **データ保護**: 個人情報は含まれない
- **アクセス制御**: 読み取り専用

#### 1.4.4 ブラウザ対応要件
- **必須対応**: Chrome 90+, Firefox 88+, Safari 14+, Edge 90+
- **推奨**: Chrome 最新版
- **レスポンシブ対応**: スマートフォン・タブレット対応

## 2. 機能要件（現状実装済み）

### 2.1 表示機能

#### 2.1.1 集計サマリー表示
- **対象会場数**: 予測対象となった会場数
- **予測レース数**: 予測対象となったレース数
- **成功レース数**: 予測が成功したレース数
- **実行時間**: 予測処理にかかった時間（分）
- **予測日**: 予測対象の日付
- **生成日時**: 予測結果の生成日時
- **モデル情報**: 使用したモデル名・バージョン
- **学習データ期間**: モデル学習に使用したデータの期間

#### 2.1.2 フィルター機能
- **会場フィルター**: 全会場/個別会場選択
- **リスクレベルフィルター**: 低/中/高リスク選択

#### 2.1.3 レース詳細表示
- **レース基本情報**: 会場名、レース番号、開始時刻
- **リスクレベル**: 低/中/高リスクの表示
- **3連単上位20組**: 予測確率・期待値の表示
- **購入方法提案**: 流し・ボックス・流しの提案

### 2.2 操作性

#### 2.2.1 レスポンシブ対応
- **デスクトップ**: フル機能表示
- **タブレット**: 主要機能表示
- **スマートフォン**: 基本機能表示

#### 2.2.2 インタラクション
- **レース詳細の開閉**: クリックで詳細表示/非表示
- **フィルター適用**: リアルタイムで結果更新
- **ローディング表示**: データ読み込み中の表示
