# サイトから取得できるデータと現状の処理

**目的**: boatrace.jp 公式サイトで取得可能なデータを一覧化し、現状の処理で拾えているか・拾えていないかを整理する。  
**前提**: 最終的には**レース前に取得できる情報**を使って予測する。

**最終更新**: 2025-02-12

---

## 1. 公式サイトのページと取得可能データ

### 1.1 出走表（racelist / entry_page）

| URL | 取得タイミング | データ項目 | 現状の取得 |
|-----|----------------|------------|------------|
| `/race/racelist?rno=&jcd=&hd=` | 締切前〜当日 | **race_info**: 日付・会場・レース番号・タイトル・締切時刻・周回・進入固定 | ✅ 取得済み |
| 同上 | 同上 | **race_entries**: 艇番・選手名・登録番号・級別・支部・出身・生年月日・身長・性別 | ✅（選手名は失敗時「不明」） |
| 同上 | 同上 | **performance**: 全国勝率・当地勝率 | ✅ |
| 同上 | 同上 | **boat**: 艇番・連対率・3連対率 | ✅ |
| 同上 | 同上 | **motor**: モーター番号・連対率・3連対率 | ✅ |

※ 支部・出身・生年月日・身長・性別は公式ページに無い場合 null。現状も取得している。

---

### 1.2 直前情報（beforeinfo）

| URL | 取得タイミング | データ項目 | 現状の取得 |
|-----|----------------|------------|------------|
| `/race/beforeinfo?rno=&jcd=&hd=` | **展示走実施後のみ** | **締切予定時刻**（表上部） | △ 出走表の deadline_at で代用可 |
| 同上 | 同上 | **スタート展示**: 艇番・進入コース・ST（フライング負値・出遅れ1.0） | ✅ start_exhibition |
| 同上 | 同上 | **周回展示**: 艇番・展示タイム | ✅ circumference_exhibition |
| 同上 | 同上 | **選手コンディション**: 登録番号・体重・調整重量 | ✅ racer_conditions |
| 同上 | 同上 | **艇設定**: 艇番・チルト・新ペラ・部品交換 | ✅ boat_settings |
| 同上 | 同上 | **水面気象**: 天気・風速・風向・気温・水温・波高・スタンド | ✅ weather_condition |

※ 展示走前はページが空または未公開のため、取得できない。現状は `fetch_before_information` で全項目取得済み。

## 2. レース前に取得できる情報のまとめ

| 種別 | 取得可否 | 関数 | 備考 |
|------|----------|------|------|
| **出走表** | ✅ 常に取得可 | `fetch_race_entry_data` | 締切前〜当日 |
| **直前情報** | ✅ 展示走後のみ | `fetch_before_information` | スタート展示・周回展示・体重・艇設定・天候 |
| **3連単オッズ** | ✅ 締切後のみ | `fetch_trifecta_odds` | 締切前は未公開 |

予測時は「出走表」を必須とし、「直前情報」は取得できた場合にマージして利用する。「オッズ」は締切後に取得して期待値・回収率計算に使う。

---

## 3. 現状の処理で拾えていないもの

- **出走表・直前情報・オッズ**: いずれも**取得処理は実装済み**。拾えていない項目はなし。
- **レース前の一本化**: 出走表＋直前情報をまとめて返す「レース前用統合取得」がなかったため、**`fetch_pre_race_data`** を追加し、予測ツールから利用する。

---

## 4. 実装上の対応（実施済み・実施予定）

| 対応 | 内容 |
|------|------|
| **fetch_pre_race_data 追加** | 出走表＋直前情報をマージして返す。直前情報が未公開でも出走表だけで返す。 |
| **予測ツールでの利用** | 本番取得時は `fetch_pre_race_data` を使い、取得した race_data に直前情報を含める。 |
| **状態ベクトル** | 現行の 192 次元は出走表＋オッズのみ。直前情報（ST・展示タイム・天候等）は将来の特徴量拡張で利用可能（データ構造は用意済み）。 |
