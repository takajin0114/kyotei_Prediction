# 学習デバッグ計画

## 現状分析

### 学習結果
- **全試行で平均報酬: -100.0**
- **エラーは発生せず、学習は完了**
- **データ的中率: 100%（全10021レースで的中action存在）**

### 問題の特定
データは正常だが、モデルが一度も的中を予測できていない = **学習が全く進んでいない**

## 実行済みタスク

### ✅ タスク1.1: 現在の報酬設計の確認（完了）

#### 実行結果
- **的中率**: 0.8333% (100/12000アクション)
- **報酬分布**:
  - 不的中時: -100 (99.17%)
  - 的中時: 平均10,327円、最大202,200円
  - 報酬の分散: 非常に大きい（標準偏差25,425円）

#### 問題点の特定
1. **報酬の疎性**: 的中率が0.83%と非常に低く、学習初期に報酬が得られにくい
2. **報酬のスケール**: 的中時と不的中時の報酬差が極端（-100 vs +10,000+）
3. **学習困難性**: 初期探索で報酬が得られず、学習が進まない

### ✅ タスク2.1: 入力データの分布確認（完了）

#### 実行結果
- **状態ベクトル**: 192次元、形状(100, 192)
- **データ品質**: 基本的に良好、欠損値は適切に処理済み
- **特徴量の正規化**: 0-1に正規化済み

#### 問題のある特徴量
- `boat0_pit`, `boat3_pit`, `boat5_pit` - 分散0（固定値）
- `laps` - 分散0（固定値）
- `is_fixed` - 分散0（固定値）

#### 問題点の特定
1. **特徴量の多様性不足**: 一部の特徴量が固定値で学習に寄与しない
2. **相関行列の計算エラー**: 分散0の特徴量による数値計算エラー

### ✅ タスク5.1: 簡易ケースでの学習テスト（完了）

#### 実行結果
- **学習成功**: エラーなしで完了
- **簡易テスト結果**: 10エピソード中2回的中、平均報酬254.00
- **ランダムアクション**: 平均-46.55、標準偏差956.14、不的中99.2%

#### 重要な発見
- **環境の基本動作は正常**: 観測空間、アクション空間、報酬計算は正常
- **学習アルゴリズムは動作**: PPOは正常に学習を実行
- **問題は報酬設計**: 本番データ・長期学習での報酬の疎性が課題

## 根本原因の特定

### 主要な問題
1. **報酬設計の不適切性**
   - 的中率が0.83%と極めて低い
   - 報酬の分散が大きすぎる（-100 vs +10,000+）
   - 学習初期に報酬が得られにくい

2. **特徴量の多様性不足**
   - 一部の特徴量が固定値
   - 学習に寄与しない特徴量の存在

3. **学習設定の課題**
   - 学習時間が短い（5,000ステップ）
   - 探索が不十分

## 次ステップ詳細タスク

### Phase 1: 報酬設計の見直し

#### タスク1.2: 段階的報酬設計の実装（優先度: 高）
- **目的**: より細かい報酬設計で学習を促進
- **実施内容**:
  - 部分的中の報酬設計（1着的中、2着的中など）
  - 順位予測の報酬設計
  - 負の報酬の調整（-100以外のペナルティ）
- **期待成果**: 学習初期から報酬が得られる設計

### Phase 2: 状態ベクトルの確認

#### タスク2.2: 特徴量エンジニアリングの改善（優先度: 中）
- **目的**: より予測に適した特徴量の作成
- **実施内容**:
  - 分散0の特徴量の除去または改善
  - 新しい特徴量の追加（過去成績、相性など）
  - 特徴量選択の最適化
- **期待成果**: 予測精度向上のための特徴量改善

### Phase 3: モデル構造の見直し

#### タスク3.1: 現在のモデル構造の確認（優先度: 低）
- **目的**: ニューラルネットワークの構造が適切か確認
- **実施内容**:
  - 現在のネットワーク構造の分析
  - 層数、ニューロン数、活性化関数の評価
  - 入力・出力層の設計確認
- **期待成果**: モデル構造の問題点特定

#### タスク3.2: モデル構造の最適化（優先度: 低）
- **目的**: より適切なモデル構造の実装
- **実施内容**:
  - 異なるネットワーク構造のテスト
  - アテンション機構の導入検討
  - ドロップアウト・バッチ正規化の調整
- **期待成果**: 学習効率向上のための構造改善

### Phase 4: 学習設定の調整

#### タスク4.1: ハイパーパラメータの最適化（優先度: 中）
- **目的**: より適切な学習パラメータの設定
- **実施内容**:
  - 学習率の調整（現在の範囲確認）
  - バッチサイズの最適化
  - エポック数の増加
  - その他のPPOパラメータ調整
- **期待成果**: 学習効率の向上

#### タスク4.2: 学習時間の延長（優先度: 中）
- **目的**: 十分な学習時間の確保
- **実施内容**:

  - 評価頻度の調整
  - 早期停止条件の設定
- **期待成果**: 学習の収束確認

### Phase 5: デバッグ用テスト
