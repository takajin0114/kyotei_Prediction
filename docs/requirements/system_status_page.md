# システムステータスページ要件・UI設計

**最終更新日**: 2025-07-15  
**バージョン**: 1.0

---

## 🎯 目的
- バッチ処理・データ取得・予測システムの稼働状況やエラー、進捗をWeb上で可視化し、運用監視・トラブル対応を容易にする

## 👤 対象ユーザー
- システム運用者（現状は開発者本人）
- 将来的な運用担当者

---

## 🖥️ 画面イメージ（ワイヤーフレーム案）

- **ヘッダー**: 「システムステータス」タイトル、更新ボタン
- **サマリーカード**: 稼働中/停止中、バッチ進捗率、エラー件数、最終実行時刻
- **進捗グラフ**: 日別・会場別の進捗バー
- **エラーログ一覧**: 直近のエラー内容・発生時刻・種別
- **バッチ履歴**: 実行履歴テーブル（日時・会場・件数・結果）
- **モデル情報**: モデル名・学習期間・精度・最終更新
- **データ品質サマリー**: 欠損・異常件数、最新チェック日

---

## 📋 表示項目一覧

| 項目             | 内容例/説明                         |
|------------------|-------------------------------------|
| システム状態     | 稼働中/停止中/警告                  |
| バッチ進捗率     | 例: 744/4152 (17.9%) 完了           |
| エラー件数       | 例: 2件（直近24h）                  |
| 最終実行時刻     | 例: 2025-07-15 22:11                |
| 進捗グラフ       | 日別・会場別のバーグラフ             |
| エラーログ       | 発生時刻・内容・種別                 |
| バッチ履歴       | 日時・会場・件数・結果               |
| モデル情報       | モデル名・学習期間・精度             |
| データ品質       | 欠損件数・異常件数・最新チェック日   |

---

## 🗂️ データ構造・取得元

- **バッチ進捗**: ログファイル/進捗JSON
- **エラーログ**: エラーログファイル/DB
- **バッチ履歴**: 実行履歴JSON/DB
- **モデル情報**: モデル管理JSON/DB
- **データ品質**: quality_report_*.json

---

## 🔧 技術要件
- Bootstrap 5ベースの管理画面UI
- JavaScriptによる非同期データ取得・自動更新
- JSON/ファイルベースのデータ取得（初期）
- 将来的なAPI化を見据えた設計
- レスポンシブ対応

---

## 🛠️ 運用・拡張要件
- 手動/自動リロード対応
- エラー発生時のアラート表示
- ログ・履歴のダウンロード機能（将来）
- 管理者のみアクセス可能な設計（将来）
