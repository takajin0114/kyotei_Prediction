# Phase 4: 拡張・UX改善 要件定義・UI設計

## 概要

Phase 4では、予測表示画面・システムステータス画面のユーザビリティ向上と拡張機能を追加します。主な目的は、データの検索・ソート・エクスポート、グラフ表示、履歴比較、ユーザー設定保存機能の実装です。

**実装開始日**: 2025-07-15  
**予定完了日**: 2025-07-24  
**優先度**: 高（WANT/NICE TO HAVE）

---

## 1. 機能要件

### 1.1 検索・フィルター機能

#### 対象画面
- **予測表示画面**（メイン対象）
- **システムステータス画面**（一部機能）

#### 検索項目
- **会場名**: ドロップダウン選択（全24会場）
- **日付**: 日付ピッカー（過去30日間）
- **リスクレベル**: ラジオボタン（低・中・高・全て）
- **レース番号**: テキスト入力（1-12）
- **艇番**: テキスト入力（1-6）
- **選手名**: テキスト入力（部分一致）

#### フィルター機能
- **的中/不的中**: チェックボックス（的中のみ・不的中のみ・全て）
- **高期待値**: チェックボックス（期待値1.5以上のみ）
- **オッズ取得済み**: チェックボックス（オッズデータありのみ）

#### UI配置
```
[検索バー]
会場名: [ドロップダウン] 日付: [日付ピッカー] リスク: [ラジオボタン]
レース番号: [入力] 艇番: [入力] 選手名: [入力]
[フィルター]
□ 的中のみ □ 高期待値のみ □ オッズ取得済み [検索] [クリア]
```

### 1.2 ソート機能

#### 対象画面
- **予測表示画面**（メイン対象）

#### ソート項目
- **確率**: 降順/昇順（デフォルト: 降順）
- **期待値**: 降順/昇順（デフォルト: 降順）
- **オッズ**: 降順/昇順
- **レース番号**: 昇順/降順
- **会場名**: 昇順/降順
- **日付**: 昇順/降順

#### UI実装
- テーブルヘッダークリックで昇順/降順切替
- ソートアイコン表示（↑↓）
- 現在のソート状態を視覚的に表示

### 1.3 エクスポート機能

#### 対象画面
- **予測表示画面**（メイン対象）
- **システムステータス画面**（一部機能）

#### エクスポート形式
- **CSV**: 基本データ形式（優先実装）
- **JSON**: 完全データ形式（優先実装）
- **PDF**: レポート形式（将来実装）

#### エクスポート内容
- **予測表示画面**: 表示中の予測データ（フィルター適用後）
- **システムステータス画面**: バッチ履歴・エラーログ

#### UI配置
```
[エクスポート]
形式: [ドロップダウン: CSV/JSON] [エクスポート] [ダウンロード中...]
```

### 1.4 グラフ・チャート表示

#### 対象画面
- **予測表示画面**（新規タブ追加）

#### グラフ内容
- **確率分布ヒストグラム**: 予測確率の分布
- **期待値分布ヒートマップ**: 期待値の分布
- **的中率推移グラフ**: 日別的中率の推移
- **会場別的中率比較**: 会場ごとの的中率

#### 実装技術
- **Chart.js**: メインのグラフライブラリ
- **レスポンシブ対応**: モバイルでも表示可能
- **インタラクティブ**: ホバー・クリックでの詳細表示

#### UI配置
```
[タブ]
予測表示 | グラフ分析 | 履歴比較
```

### 1.5 履歴比較・複数日付比較

#### 対象画面
- **予測表示画面**（新規タブ追加）

#### 比較機能
- **日付比較**: 任意の2日間の予測結果を並列表示
- **会場比較**: 同一日付の複数会場を比較
- **的中率比較**: 過去の予測と実績の比較
- **期待値比較**: 期待値の推移分析

#### UI配置
```
[比較設定]
比較対象: [日付1] vs [日付2] [比較実行]
[比較結果]
左: 日付1の予測結果 | 右: 日付2の予測結果
```

### 1.6 ユーザー設定保存・通知

#### 設定保存機能
- **検索条件**: 会場名・日付・リスクレベル
- **ソート順**: 現在のソート設定
- **表示項目**: 表示/非表示の列設定
- **フィルター設定**: 適用中のフィルター

#### 保存方法
- **ローカルストレージ**: ブラウザ内保存
- **設定ファイル**: JSON形式でのエクスポート/インポート

#### 通知機能（将来実装）
- **新着データ**: 新しい予測データの通知
- **エラー通知**: システムエラーの通知
- **バッチ完了**: データ取得完了の通知

## 2. 非機能要件

### 2.1 パフォーマンス
- **レスポンス時間**: 検索・ソートは1秒以内
- **大量データ対応**: 1000件以上のデータでも快適に動作
- **メモリ使用量**: ブラウザメモリを効率的に使用

### 2.2 レスポンシブ対応
- **デスクトップ**: 1200px以上（フル機能）
- **タブレット**: 768px - 1199px（主要機能）
- **モバイル**: 576px - 767px（基本機能）
- **小型モバイル**: 575px以下（最小限機能）

### 2.3 アクセシビリティ
- **キーボード操作**: 全機能をキーボードで操作可能
- **スクリーンリーダー**: 適切なARIA属性とラベル
- **色覚対応**: 色だけでなく形状でも情報を表現
- **フォントサイズ**: ブラウザ設定に応じた拡大縮小対応

### 2.4 セキュリティ
- **XSS対策**: 適切なHTMLエスケープ処理
- **CSRF対策**: トークンベースの認証（将来実装）
- **ダウンロード安全性**: ファイル名・パスの適切な処理

## 5. 技術仕様

### 5.1 フロントエンド技術
- **HTML5**: セマンティックなマークアップ
- **CSS3**: レスポンシブデザイン・アニメーション
- **JavaScript ES6+**: モジュラー設計・クラスベース
- **Bootstrap 5**: UIフレームワーク
- **Chart.js**: グラフライブラリ

### 5.2 データ管理
- **ローカルストレージ**: ユーザー設定保存
- **JSON**: データ形式・設定ファイル
- **CSV**: エクスポート形式

### 5.3 パフォーマンス最適化
- **仮想スクロール**: 大量データの効率的表示
- **遅延読み込み**: 必要時のみデータ読み込み
- **キャッシュ**: 検索結果の一時保存
