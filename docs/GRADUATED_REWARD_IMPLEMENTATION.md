# 段階的報酬設計の実装

## 概要

従来の報酬設計では的中時のみ報酬が与えられ、学習の疎性が問題となっていました。段階的報酬設計を実装することで、部分的中にも報酬を与え、学習の効率性を大幅に改善しました。

## 実装内容

### 1. 報酬設計の変更

**旧報酬設計:**
- 的中時: 払戻金 - 賭け金
- 不的中時: -賭け金（-100）

**新報酬設計（段階的）:**
- 的中時: 払戻金 - 賭け金
- 2着的中時: -10（1着、2着が正解）
- 1着的中時: -50（1着のみ正解）
- 不的中時: -100（全く的中なし）

### 2. 実装ファイル

- `kyotei_predictor/pipelines/kyotei_env.py`: `calc_trifecta_reward`関数を修正

## 実装の詳細

### 報酬計算関数の修正

```python
def calc_trifecta_reward(action: int, arrival_tuple: Tuple[int,int,int], odds_data: list, bet_amount: int = 100) -> float:
    """
    action（0-119）, 着順タプル, oddsデータ, 賭け金を受け取り、段階的報酬を返す。
    
    段階的報酬設計:
    - 的中時: 払戻金-賭け金
    - 2着的中時: -10（1着、2着が正解）
    - 1着的中時: -50（1着のみ正解）
    - 不的中時: -100（全く的中なし）
    """
    trifecta = action_to_trifecta(action)
    
    # 的中判定
    is_win = trifecta == arrival_tuple
    
    if is_win:
        # 的中時: 払戻金-賭け金
        odds_map = {tuple(o['betting_numbers']): o['ratio'] for o in odds_data}
        odds = odds_map.get(trifecta, 0)
        payout = odds * bet_amount
        reward = payout - bet_amount
    else:
        # 部分的中の判定
        first_hit = trifecta[0] == arrival_tuple[0]  # 1着的中
        second_hit = trifecta[1] == arrival_tuple[1]  # 2着的中
        
        if first_hit and second_hit:
            # 2着的中: 1着、2着が正解
            reward = -10
        elif first_hit:
            # 1着的中: 1着のみ正解
            reward = -50
        else:
            # 不的中: 全く的中なし
            reward = -100
    
    return reward
```
