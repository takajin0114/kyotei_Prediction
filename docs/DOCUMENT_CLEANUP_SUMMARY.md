## 🗂️ ドキュメント整理方針（2025-07-15時点）

### 1. 進捗・現状サマリー系の統合・役割明確化
- `CURRENT_STATUS_SUMMARY.md`：プロジェクト全体の現状・進捗サマリーに特化。詳細なTODOや直近の優先タスクは「NEXT_STEPS.md」へ集約。
- `NEXT_STEPS.md`：直近のTODO・優先度・進捗管理に特化。完了済み・進行中・今後のタスクを一元管理。
- 各機能ごとの「現状・進捗・課題」は該当機能の設計/要件/計画ドキュメントの末尾に「現状・進捗」セクションを設けて集約。
- `DOCUMENT_CLEANUP_SUMMARY.md`はドキュメント運用履歴・削除/統合履歴のみを記載。

### 2. 設計・要件・計画系の整理
- Web表示機能関連（`web_display_requirements.md`, `WEB_DISPLAY_IMPLEMENTATION_PLAN.md`, `WEB_DISPLAY_IMPLEMENTATION_COMPLETE.md`）は1つの「Web表示機能」ディレクトリ（例：`docs/web_display/`）にまとめ、要件・計画・完了報告を整理。
- 画面・機能ごとの要件は「要件定義」ディレクトリ（例：`docs/requirements/`）に集約。
- `DEVELOPMENT_ROADMAP.md`は全体の長期計画・マイルストーンのみ、各機能の詳細な計画は該当機能のドキュメントに記載。

### 3. 運用・技術ガイド系の整理
- 運用ガイド・実行例は「運用ガイド」ディレクトリ（例：`docs/operations/`）に集約。
- `DOCUMENTATION_STANDARDS.md`はドキュメント運用ルールに一元化し、運用履歴や整理履歴は「DOCUMENT_CLEANUP_SUMMARY.md」へ。

### 4. README.mdの役割
- `README.md`はドキュメント索引・プロジェクト概要・クイックスタート・進捗まとめに特化し、各詳細ドキュメントへのリンクを最新化。
