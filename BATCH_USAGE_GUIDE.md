# 🚀 競艇予測最適化 バッチ実行ガイド

## 📋 概要

このガイドでは、競艇予測最適化の実行を自動化するバッチファイルの使用方法について説明します。

## 🎯 作成されたバッチファイル

### 1. **run_optimization_batch.bat** - 統合実行バッチ
- **機能**: 最適化実行、ログ出力、仮想環境化、削除処理を一元化
- **特徴**: 設定変更なしで即座に実行可能
- **用途**: 定期的な最適化実行、テスト実行

### 2. **run_optimization_config.bat** - 設定可能実行バッチ
- **機能**: 設定ファイルでパラメータを管理
- **特徴**: 実行前に設定変更・確認が可能
- **用途**: 柔軟な最適化実行、異なるパラメータでの実行

### 3. **cleanup_old_files.bat** - クリーンアップ専用バッチ
- **機能**: 古いファイルの削除のみ
- **特徴**: 最適化実行なしでクリーンアップのみ実行
- **用途**: ディスク容量の確保、定期メンテナンス

## 🚀 使用方法

### **基本的な実行手順**

#### **1. 統合実行バッチ（推奨）**
```bash
# ダブルクリックまたはコマンドラインで実行
run_optimization_batch.bat
```

**実行内容**:
- 仮想環境の確認・作成
- 依存関係の確認・インストール
- 高速モードで20試行の最適化実行
- 実行時間の記録
- 古いファイルの自動クリーンアップ
- 詳細なログ出力

#### **2. 設定可能実行バッチ**
```bash
# ダブルクリックまたはコマンドラインで実行
run_optimization_config.bat
```

**実行内容**:
- 設定ファイルの読み込み
- 設定値の表示・変更
- 実行確認
- 最適化実行
- クリーンアップ処理

#### **3. クリーンアップ専用**
```bash
# ダブルクリックまたはコマンドラインで実行
cleanup_old_files.bat
```

**実行内容**:
- 削除対象日数の設定
- 古いファイルの削除
- 空ディレクトリの削除

## ⚙️ 設定ファイルの編集

### **optimization_config.ini**

```ini
# 最適化モード (fast: 高速モード, normal: 通常モード)
MODE=fast

# 試行回数
TRIALS=20

# 対象年月 (YYYY-MM形式、空の場合は全期間)
YEAR_MONTH=2024-01

# 仮想環境のパス
VENV_PATH=venv

# ログディレクトリ
LOG_DIR=logs

# クリーンアップ対象の日数（古いファイルを削除）
CLEANUP_DAYS=7
```

### **設定変更の例**

#### **高速モードで50試行実行**
```ini
MODE=fast
TRIALS=50
YEAR_MONTH=2024-01
```

#### **通常モードで100試行実行**
```ini
MODE=normal
TRIALS=100
YEAR_MONTH=2024-02
```

#### **全期間データで実行**
```ini
MODE=fast
TRIALS=30
YEAR_MONTH=
```

## 📊 ログ出力

### **ログファイルの場所**
- **ディレクトリ**: `logs/`
- **ファイル名**: `optimization_YYYYMMDD_HHMM.log`
- **例**: `optimization_20250817_143022.log`

### **ログに記録される情報**
- 実行開始・終了時刻
- 最適化の詳細出力
- エラー・警告メッセージ
- 実行結果サマリー

## 🧹 クリーンアップ機能

### **自動クリーンアップ対象**
- **Optuna関連ファイル**: 7日以上古いもの
- **ログファイル**: 30日以上古いもの
- **対象ディレクトリ**:
  - `optuna_logs/`
  - `optuna_models/`
  - `optuna_results/`
  - `optuna_studies/`
  - `optuna_tensorboard/`

### **クリーンアップのタイミング**
- 最適化実行後の自動実行
- 手動実行（`cleanup_old_files.bat`）

## 🔧 トラブルシューティング

### **よくある問題と解決方法**

#### **1. 仮想環境が見つからない**
```
❌ 仮想環境が見つかりません: venv
```
**解決方法**: バッチファイルが自動的に仮想環境を作成します

#### **2. 依存関係の不足**
```
❌ 必要なパッケージが不足しています
```
**解決方法**: バッチファイルが自動的に`requirements.txt`からインストールします

#### **3. 権限エラー**
```
❌ ファイルの削除に失敗しました
```
**解決方法**: 管理者権限で実行するか、該当ファイルを手動で削除

#### **4. ログファイルが作成されない**
**解決方法**: `logs/`ディレクトリの存在確認、書き込み権限の確認


## 🎉 実行例

### **基本的な実行フロー**
1. バッチファイルをダブルクリック
2. 設定確認・変更（設定可能バッチの場合）
3. 実行確認
4. 最適化実行
5. 結果表示
6. クリーンアップ処理
7. 完了

### **実行時間の目安**
- **高速モード 20試行**: 10-15分
- **高速モード 50試行**: 25-40分
- **高速モード 100試行**: 50-80分
- **通常モード 100試行**: 8-12時間

4. 依存関係のインストール状況
